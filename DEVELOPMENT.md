# Development Guide — scud-astro-web

## 📦 Prerequisites

- **Node**: >=22.12.0
- **Package Manager**: pnpm (recommended)
- **VS Code Extensions**: See `.vscode/extensions.json` (recommended)

## 🚀 Setup

### Initial Setup

```bash
# Install dependencies
pnpm install

# Start development server
pnpm dev
# Open http://localhost:4321
```

### Verify Setup

```bash
# Check Node version
node --version  # Should be >=22.12.0

# Check pnpm version
pnpm --version

# Run build (should succeed with no errors)
pnpm build
```

| Command | Purpose | Output |
|---------|---------|--------|
| `pnpm dev` | Start dev server | Local: http://localhost:4321 |
| `pnpm build` | Build for production | `dist/` folder |
| `pnpm preview` | Preview production build | http://localhost:3000 |

## 🌍 i18n Architecture

### URL Structure

| URL | Locale | Description |
|-----|--------|-------------|
| `/` | `es` | Spanish (default) — no prefix |
| `/ca/` | `ca` | Catalan — `/ca/` prefix |
| `/en/` | `en` | English — `/en/` prefix |
| `/about` | `es` | Spanish about page |
| `/ca/about` | `ca` | Catalan about page |
| `/en/about` | `en` | English about page |

### Translation Files Structure

Each JSON file (`es.json`, `ca.json`, `en.json`) has this structure:

```json
{
  "nav": {
    "home": "Inicio",
    "about": "Acerca de",
    "services": "Servicios"
  },
  "cta": {
    "primary": "Comenzar",
    "secondary": "Más información"
  },
  "footer": {
    "copyright": "© 2026 Scud Security"
  },
  "routes": {
    "home": "/",
    "about": "/about/",
    "services": "/services/"
  }
}
```

### Key Functions (`src/utils/i18n.ts`)

```typescript
// Get locale from URL
const locale = getLocaleFromPath("/ca/about");  // "ca"

// Get locale in component
const locale = getCurrentLocale(Astro);  // "es", "ca", or "en"

// Access translations
const i18n = getI18n("es");
console.log(i18n.nav.home);  // "Inicio"
```

## 🛠️ Workflows

### Adding a New Page (Multiidioma)

#### **Recommended: Using the Skill**

Tell Copilot: *"Add a new page called 'Contact' in Spanish, Catalan, and English"*

The `scud-astro-add-page` skill automates:
- Creating 3 `.astro` files (es, ca, en) with Layout template
- Generating base translation keys in all 3 JSON files
- Validating file structure and key synchronization

#### **Manual Process** (if not using skill)

**Step 1**: Create 3 page files

```bash
# Spanish version
touch src/pages/contacto.astro

# Catalan version
touch src/pages/ca/contacto.astro

# English version
touch src/pages/en/contacto.astro
```

**Step 2**: Add content to each file

```astro
---
// src/pages/contacto.astro
import Layout from "../layouts/Layout.astro";
import { getCurrentLocale, getI18n } from "../utils/i18n";

const locale = getCurrentLocale(Astro);
const i18n = getI18n(locale);
---

<Layout>
  <main>
    <h1>{i18n.contact.title}</h1>
    <p>{i18n.contact.description}</p>
  </main>
</Layout>
```

**Step 3**: Add translations to all 3 JSON files

```json
// src/i18n/es.json
{
  "contact": {
    "title": "Contacto",
    "description": "Ponte en contacto con nosotros..."
  }
}

// src/i18n/ca.json
{
  "contact": {
    "title": "Contacte",
    "description": "Poseu-vos en contacte amb nosaltres..."
  }
}

// src/i18n/en.json
{
  "contact": {
    "title": "Contact",
    "description": "Get in touch with us..."
  }
}
```

**Step 4** (Optional): Update Navigation

If you want the page to appear in the navigation menu, update `src/i18n/{es,ca,en}.json` routes section and update `src/components/ui/Navigation.astro` if needed.

### Adding a Component with i18n

```astro
---
// src/components/MyComponent.astro
import { getCurrentLocale, getI18n } from "../utils/i18n";

const locale = getCurrentLocale(Astro);
const i18n = getI18n(locale);
---

<section>
  <h2>{i18n.myComponent.title}</h2>
  <p>{i18n.myComponent.description}</p>
</section>
```

### Adding a Translation Key

1. Open `src/i18n/es.json` → add your key
2. Open `src/i18n/ca.json` → add your key with Catalan translation
3. Open `src/i18n/en.json` → add your key with English translation

**Important**: Keep the JSON structure synchronized. If you add `footer.newKey` in one file, add it to all three.

### Updating Global Styles

Edit `src/styles/global.css`:

```css
@import "tailwindcss";

/* Your custom globals here */
:root {
  --font-lato: Lato, system-ui, sans-serif;
}
```

Tailwind is integrated via `@tailwindcss/vite` plugin (see `astro.config.mjs`).

## 🐛 Troubleshooting

### Issue: Component not recognizing translation key

**Symptom**: `TypeError: Cannot read property 'title' of undefined`

**Solution**:
1. Check that the key exists in `src/i18n/{locale}.json`
2. Verify JSON syntax (no trailing commas)
3. Verify file is saved
4. Restart dev server: `Ctrl+C` → `pnpm dev`

### Issue: Build fails with locale error

**Symptom**: `Error: Invalid locale 'xx' — expected one of: es, ca, en`

**Solution**:
1. Check that page files exist for all 3 locales
2. Verify `astro.config.mjs` has correct locale list
3. Check file paths are correct (e.g., `pages/ca/`, not `pages/ca-CA/`)

### Issue: Prettier not formatting on save

**Symptom**: Code remains unformatted

**Solution**:
1. Install Prettier extension: `esbenp.prettier-vscode`
2. Check `.vscode/settings.json` has `"editor.formatOnSave": true`
3. Reload VS Code window (Cmd+Shift+P → "Developer: Reload Window")

### Issue: TypeScript errors in components

**Symptom**: Red squiggles, IntelliSense not working

**Solution**:
1. Check VS Code is using workspace TypeScript: Cmd+Shift+P → "TypeScript: Select TypeScript Version" → "Use Workspace Version"
2. Verify `tsconfig.json` exists and is valid
3. Restart TS server: Cmd+Shift+P → "TypeScript: Restart TS Server"

### Issue: Tailwind classes not recognized

**Symptom**: Styling not applied, no IntelliSense for Tailwind classes

**Solution**:
1. Install Tailwind CSS extension: `bradlc.vscode-tailwindcss`
2. Check `tailwind.config.ts` has correct content paths
3. Verify `astro.config.mjs` includes Tailwind plugin
4. Reload VS Code window

### Before Committing

1. Run `pnpm build` — ensure build succeeds
2. Check format: `pnpm dev` — all pages load without errors
3. Review commit message format in `.github/instructions/copilot-commit-message.md`

### Commit Message Format

```
<type>(<scope>): <description>

Examples:
feat(pages): add contact page multiidioma
fix(i18n): sync missing translation keys
refactor(components): simplify Navigation component
```

**Valid types**: `feat`, `fix`, `refactor`, `perf`, `docs`, `style`, `test`, `build`, `ci`, `chore`, `revert`
